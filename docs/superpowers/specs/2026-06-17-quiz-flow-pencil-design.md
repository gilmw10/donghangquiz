# 체험확인 퀴즈 — 전체 흐름 Pencil 디자인 스펙

**날짜:** 2026-06-17
**대상:** 서울디지털동행플라자 체험확인 퀴즈 시스템
**범위:** 메인 → 문제(O/X) → 결과 → 쿠폰 모달 전체 흐름의 시각 디자인 (`pencil-new.pen`)
**참고:** https://didong.kr/ 의 디자인 느낌 (깔끔한 흰 배경, 파랑/청록 포인트, 카드 기반, 넉넉한 여백, 노인 친화 접근성)

---

## 1. 목적

디지털 기기에 익숙하지 않은 노인 이용자가 교육·체험 후 내용을 O/X 퀴즈로 확인하는 웹앱.
주제 3개 중 하나를 골라 10문제를 풀고, 통과하면 도장을 받는다. 3개 주제를 모두
통과하면 커피 쿠폰을 받는다.

---

## 2. 기준 사양

- **화면 비율:** 16:10, 데스크톱 웹 기준 **1440 × 900**
- **도구:** Pencil (`.pen`) — 시각 디자인. (React 구현은 별도)
- **언어:** 한국어 UI

---

## 4. 점수 체계

- 10문제 = 일반 8문제(각 **100점**) + 고난도 2문제(각 **300점**)
- 고난도는 무조건 2문제 출제 → **만점 1400점**
- **1000점 이상** → "잘하셨어요! 통과하셨습니다!" → 해당 주제 **도장 획득**
- **1000점 미만** → "아쉬워요! 다시한번 체험해보고 올까요?"
- 3개 주제 모두 도장 → **커피 쿠폰 모달**

---

## 5. 화면 구성 (6 프레임)

### 5-1. 메인 (주제 선택)
- 파란 헤더: 아이콘 + "서울디지털동행플라자" + "디지털 기기, 이제 자신있게!" / 우측 "체험확인 퀴즈" 뱃지
- 페이지 타이틀 "주제를 선택하세요" + 안내문 + 진행도 "3개 중 N개 완료"
- TopicCard ×3 — 좌측 컬러 바, 주제 아이콘, 제목, 세부항목, "시작하기" 버튼
  - **도장 상태:** 통과한 주제는 빨간 인장 느낌 "통과" 도장 오버레이
- 푸터

### 5-2. 문제 (O/X) — 일반 / 고난도 변형
- 주제색 헤더 + 진행 표시 "문제 N / 10" (점 또는 바)
- **배점 뱃지:** 일반 "일반 · 100점", 고난도 "고난도 · 300점"(강조)
- 큰 문제 텍스트(상황 설명) + 필요시 아이콘
- 큰 버튼 2개: ⭕ "맞아요" / ❌ "아니에요" (넓은 클릭 영역, 차분한 색 구분)
- 푸터

### 5-3. 결과 — 통과 / 아쉬움 변형
- **통과(≥1000):** "잘하셨어요! 통과하셨습니다!" + 점수(예 1200/1400) + 도장 획득 연출 + "다른 주제 선택하기"
- **아쉬움(<1000):** "아쉬워요! 다시한번 체험해보고 올까요?" + 점수(예 700/1400) + "다시 풀기"
- 격려 중심 — 틀린 문제 강조하지 않음

### 5-4. 커피 쿠폰 모달
- 메인 위 오버레이(딤 배경)
- 3개 도장이 모두 찍힌 모습 + "축하합니다! 커피 쿠폰을 드려요" + 쿠폰 카드 + "받기 / 닫기" 버튼

## 6. 공통 컴포넌트 (Pencil reusable)

헤더 · 푸터 · TopicCard(도장 상태 포함) · O/X 버튼 · 배점 뱃지

## 7. 접근성 / 노인 친화 원칙

- 본문 최소 14px, 문제 텍스트 크게(24px+), 카드 제목 20px+
- 버튼 클릭 영역 넉넉히
- 고대비 색상, 그라디언트·글로우 최소화
- 이모티콘 남용 금지 — 아이콘으로 대체
