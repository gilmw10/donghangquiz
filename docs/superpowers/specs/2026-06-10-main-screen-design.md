# 체험확인 퀴즈 — 메인화면 디자인 스펙

**날짜:** 2026-06-10
**대상:** 서울디지털동행플라자 체험확인 퀴즈 시스템
**범위:** 메인화면(주제 선택 화면)만 — 퀴즈 화면은 별도 스펙

---

## 1. 목적

디지털 기기에 익숙하지 않은 노인 이용자들이 교육·체험 후 내용을 퀴즈로 확인할 수 있는 웹앱의 첫 진입 화면. 주제 3가지 중 하나를 선택하면 해당 퀴즈로 이동한다.

---

## 2. 기술 스택

- **프레임워크:** React (JSX)
- **스타일:** 컴포넌트별 별도 `.css` 파일 (CSS-in-JS, Tailwind, CSS Modules 사용 안 함)
- **번들러:** Vite (CRA 대신 빠른 개발 환경)
- **언어:** JavaScript (TypeScript 없음)

---

## 3. 화면 구성

### 3-1. 레이아웃 구조

```
┌─────────────────────────────────────────┐
│  AppHeader                              │  64px 고정
├─────────────────────────────────────────┤
│                                         │
│  페이지 타이틀: "주제를 선택하세요"         │
│                                         │
│  TopicCard (키오스크 사용법)              │
│  TopicCard (스마트폰 교육)               │
│  TopicCard (디지털 여가활동)              │
│                                         │
├─────────────────────────────────────────┤
│  AppFooter                              │
└─────────────────────────────────────────┘
```

### 3-2. AppHeader

- 배경: `#1d4ed8` (파란색)
- 좌측: 아이콘 + "서울디지털동행플라자" (17px bold) + "디지털 기기, 이제 자신있게!" (11px)
- 우측: "체험확인 퀴즈" 뱃지 (반투명 흰색 테두리)

### 3-3. TopicCard (×3)

각 카드는 공통 컴포넌트로, props로 데이터를 받음.

| 항목 | 값 |
|------|-----|
| 배경 | 흰색 `#fff` |
| 테두리 | `1.5px solid #e2e8f0` |
| 좌측 컬러 선 | 5px, 주제별 고유색 |
| 아이콘 | SVG, 주제별 고유색 |
| 제목 폰트 | 20px, 800 weight |
| 세부항목 폰트 | 14px, 색상 `#94a3b8` |
| 시작 버튼 | 주제별 배경색, 흰 텍스트 15px |

**주제별 색상:**

| 주제 | 색상 | 세부 항목 |
|------|------|-----------|
| 키오스크 사용법 | `#1d4ed8` (파랑) | 기차표 예약 · 은행 ATM · 음식점 주문 |
| 스마트폰 교육 | `#0891b2` (청록) | 카카오톡 · 네이버 검색 · 유튜브 |
| 디지털 여가활동 | `#7c3aed` (보라) | 무인 사진관 · 스크린 골프 · 전자칠판 |

### 3-4. AppFooter

- 배경: `#f8faff`
- 텍스트: "서울디지털동행플라자 · 체험확인 퀴즈 시스템" (12px, `#94a3b8`)

## 4. 컴포넌트 구조

```
src/
├── App.jsx                  # 라우터 없이 단순 화면 렌더
├── App.css
├── components/
│   ├── AppHeader/
│   │   ├── AppHeader.jsx
│   │   └── AppHeader.css
│   ├── AppFooter/
│   │   ├── AppFooter.jsx
│   │   └── AppFooter.css
│   └── TopicCard/
│       ├── TopicCard.jsx    # props: id, title, subtopics[], color, icon
│       └── TopicCard.css
└── pages/
    └── MainScreen/
        ├── MainScreen.jsx   # topics 데이터 정의, 카드 목록 렌더
        └── MainScreen.css
```

---

## 5. 접근성 / 노인 친화 원칙

- 최소 폰트 크기 14px (본문), 카드 제목 20px
- 버튼 클릭 영역 충분히 확보 (padding 11px 22px 이상)
- 고대비 색상 (흰 배경 위 짙은 텍스트)
- 이모티콘 사용 금지 — SVG 아이콘으로 대체
- AI 디자인 티 나는 그라디언트·글로우 효과 없음

- 점수 저장 / 결과 화면
